# 场景：电商平台限时秒杀功能

## 场景描述

在一个现有的电商平台上实现限时秒杀功能。该功能需要支持：

- 商品秒杀活动的创建、管理和查询
- 用户在秒杀时间窗口内抢购商品
- 防止超卖（库存扣减的并发控制）
- 秒杀结果实时通知
- 秒杀活动的定时启用和关闭

## 技术栈

- **后端框架**: Java Spring Boot
- **数据库**: PostgreSQL
- **缓存**: Redis
- **消息队列**: RabbitMQ
- **API 风格**: RESTful

## 为什么这个场景需要 /plan

这个场景符合使用 `/plan` 命令的所有条件：

1. **新功能开发**: 添加完整的秒杀功能模块
2. **多个技术决策**:
   - 并发控制方案（数据库锁 vs Redis Lua 脚本 vs 分布式锁）
   - 缓存策略（预热策略、更新策略）
   - 消息队列选型
3. **架构决策**: 涉及多个层次（数据库、缓存、API、消息队列）
4. **多个文件修改**: 需要创建多个实体、服务、控制器、配置文件
5. **风险和依赖**: 并发安全、性能要求、依赖其他服务
6. **实现顺序重要**: 基础设施 → 核心逻辑 → API 接口 → 定时任务 → 监控

## 使用步骤

### 第一步：启动规划

在项目根目录执行：

```bash
/plan 为电商平台实现限时秒杀功能，需要支持活动创建、用户抢购、并发控制、实时通知和定时任务
```

### 第二步：审查生成的计划

规划代理会生成详细的实现计划，包括：

- 需求重述
- 技和其他方案对比
- 分阶段实现步骤
- 依赖关系
- 风险分析
- 预估时间

### 第三步：确认或修改计划

你可以：
- 直接确认并开始实施
- 修改技术方案选择
- 调整实现步骤顺序
- 要求增加或移除某些步骤

### 第四步：按计划实施

确认后，按照计划逐步实现，可以使用 `/tdd` 进行测试驱动开发。
