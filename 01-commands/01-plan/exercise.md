# /plan 命令练习

## 练习目标

通过本练习，你将学会：
- 独立使用 `/plan` 命令创建实现计划
- 理解规划代理的工作流程
- 掌握计划审查和修改方法
- 为后续开发做好准备

## 前提条件

- 已完成 Java Spring Boot Demo 的环境搭建
- 了解基本的 REST API 概念
- 已阅读 `/plan` 命令说明文档

## 练习场景

**场景**: 为任务管理系统添加用户标签功能

**需求描述**:
- 用户可以为任务添加多个标签
- 可以按标签筛选任务
- 标签有名称和颜色属性
- 用户可以创建、编辑、删除标签
- 标签可以被多个任务使用

## 练习步骤

### 步骤 1: 使用 /plan 创建计划

在 Java Spring Boot Demo 项目中执行：

```bash
cd 04-practice/java-springboot-demo

用户: /plan 为任务管理系统添加用户标签功能，支持：
1. 用户可以为任务添加多个标签
2. 可以按标签筛选任务
3. 标签有名称和颜色属性
4. 用户可以创建、编辑、删除标签
5. 标签可以被多个任务使用
```

### 步骤 2: 审查生成的计划

规划代理会生成一个详细的实现计划。请审查以下内容：

**检查清单**:
- [ ] 需求重述是否准确
- [ ] 技术方案是否合理
- [ ] 实现步骤是否完整
- [ ] 是否包含数据库设计
- [ ] 是否包含 API 端点设计
- [ ] 是否包含测试计划
- [ ] 风险分析是否充分
- [ ] 时间估算是否合理

### 步骤 3: 修改计划（如有需要）

如果计划有任何遗漏或不准确的地方，要求修改：

```bash
用户: modify: [你的修改建议]
```

**示例修改**:
```bash
# 添加单元测试阶段
用户: modify: 在实现 API 端点之前，先添加单元测试和集成测试阶段

# 调整步骤顺序
用户: modify: 将数据库测试放到数据库实现之后

# 添加遗漏的步骤
用户: modify: 添加标签名称唯一性验证
```

### 步骤 4: 确认计划

当你对计划满意后，确认计划：

```bash
用户: yes
```

### 步骤 5: 保存计划（可选）

将生成的计划保存到项目中：

```bash
mkdir -p docs/plans
# 将计划内容保存到 docs/plans/tags-feature.md
```

### 步骤 6: 反思和总结

回答以下问题：

1. **计划生成质量**
   - 计划是否覆盖了所有需求？
   - 步骤是否清晰可行？
   - 技术方案是否合理？

2. **规划过程体验**
   - 规划过程是否顺利？
   - 有哪些地方需要改进？
   - 是否容易理解计划内容？

3. **个人学习体会**
   - 学到了什么？
   - 有哪些疑问？
   - 如何应用到实际项目？

## 练习提示

### 提示 1: 详细描述需求

```bash
# 好的做法
用户: /plan 为任务管理系统添加用户标签功能，支持：
1. 用户可以为任务添加多个标签
2. 可以按标签筛选任务
3. 标签有名称和颜色属性
4. 用户可以创建、编辑、删除标签
5. 标签可以被多个任务使用

# 不好的做法
用户: /plan 添加标签功能
```

### 提示 2: 仔细审查计划

重点关注以下方面：
- 数据模型设计是否正确（Tag 实体、Task 和 Tag 的关系）
- API 端点是否完整（CRUD 操作）
- 是否考虑了边界情况（空标签、重复标签等）
- 测试计划是否充分

### 提示 3: 不要害怕多次修改

规划是一个迭代过程。如果第一次计划不完美，可以多次修改直到满意：

```bash
用户: /plan ...
[第一次计划]
用户: modify: 添加标签颜色验证
[第二次计划]
用户: modify: 添加标签删除时的关联处理
[第三次计划]
用户: yes  # 满意了
```

### 提示 4: 保存和参考计划

将计划保存到项目中，后续开发时可以参考。这有助于：
- 保持开发方向一致
- 便于团队协作
- 方便后续回顾

## 常见问题

**Q: 计划生成太慢怎么办？**

A: 规划通常需要 1-5 分钟。如果太慢，可以尝试简化需求描述，或者检查网络连接。

**Q: 计划不符合我的预期怎么办？**

A: 要求修改计划。你可以描述你期望的计划结构，或者指出具体的问题。

**Q: 我可以自己写计划吗？**

A: 可以，但推荐使用 `/plan` 命令，因为它会基于最佳实践生成计划，并考虑项目的具体情况。

**Q: 计划必须全部实现吗？**

A: 不一定。计划是一个参考，你可以根据实际情况调整。但建议尽量遵循计划，因为它考虑了最佳实践。

**Q: 如何验证计划的质量？**

A: 审查计划的各个部分：需求重述、技术方案、实现步骤、风险分析。确保它们都准确、完整、合理。
