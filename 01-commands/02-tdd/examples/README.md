# TDD 示例集合

这个目录包含了 6 个实际的 TDD 应用场景示例，展示了测试驱动开发在不同情况下的应用。

## 📚 示例列表

### 1. [金融计算器](./01-financial-calculator.md)
**场景：** 银行利息计算
**语言：** JavaScript (Mocha/Chai)

**涵盖要点：**
- 复杂业务逻辑测试
- 边界条件处理（零值、负值）
- 小数精度计算
- 输入验证

**为什么适合 TDD：**
- 规则清晰可形式化
- 边界条件多且重要
- 金融计算容错率为零

---

### 2. [REST API 认证](./02-rest-api-authentication.md)
**场景：** 用户注册和登录 API
**语言：** Python (FastAPI + pytest)

**涵盖要点：**
- API 契约设计
- 输入验证和错误处理
- 密码强度验证
- JWT Token 生成和验证
- 受保护端点授权测试

**为什么适合 TDD：**
- API 接口需要明确定义
- 安全逻辑必须覆盖所有情况
- 多客户端集成需要可靠接口

---

### 3. [二分查找算法](./03-algorithm-binary-search.md)
**场景：** 经典算法实现
**语言：** Go (testing package)

**涵盖要点：**
- 算法正确性验证
- 表格驱动测试
- 性能基准测试
- 边界条件（空数组、单元素）
- 泛型算法抽象

**为什么适合 TDD：**
- 算法逻辑可精确验证
- 边界条件复杂且重要
- 性能要求明确可测试

---

### 4. [表单验证器](./04-form-validator.md)
**场景：** 用户注册表单验证
**语言：** TypeScript (Jest)

**涵盖要点：**
- 组合验证逻辑
- 多字段协同验证
- 错误消息本地化
- 规则配置化
- 国际化支持

**为什么适合 TDD：**
- 验证规则明确且众多
- 需要覆盖所有错误路径
- 规则变化频繁，测试确保正确性

---

### 5. [购物车领域模型](./05-domain-model-shopping-cart.md)
**场景：** 电商购物车系统
**语言：** Java (JUnit 5)

**涵盖要点：**
- 领域模型设计
- 复杂业务规则（折扣、税、运费）
- 策略模式应用
- BigDecimal 精度计算
- 不变量验证

**为什么适合 TDD：**
- 业务规则复杂且需要验证
- 多种计算策略可测试
- 领域概念清晰可建模

---

### 6. [重构遗留代码](./06-refactoring-legacy-code.md)
**场景：** 字符串工具类重构
**语言：** JavaScript

**涵盖要点：**
- 特征测试（Characterization Tests）
- 安全重构技术
- 性能基准对比
- 增量改进
- 行为保持验证

**为什么适合 TDD：**
- 重构高风险，需要安全网测试
- 需要确保行为不变
- 性能优化需要对比验证

## 🎯 使用建议

### 学习路径

1. **初学者：** 从示例 1 或 3 开始
   - 这两个场景相对独立
   - 展示 TDD 基本流程

2. **中级开发者：** 关注示例 2、4、5
   - 涉及更复杂的设计模式
   - 展示 TDD 在实际项目中的应用

3. **高级开发者：** 重点学习示例 6
   - 重构是 TDD 的重要应用场景
   - 展示如何安全地改进遗留代码

### 选择指南

| 你的需求 | 推荐示例 |
|---------|---------|
| 学习 TDD 基础流程 | 1, 3 |
| API 开发 | 2 |
| 前端表单验证 | 4 |
| 后端业务逻辑 | 1, 5 |
| 算法实现 | 3 |
| 重构遗留代码 | 6 |
| 领域驱动设计 | 5 |

### 最佳实践提取

从这些示例中可以提取的 TDD 最佳实践：

1. **测试优先** - 总是先写测试，再写代码
2. **小步前进** - 每次只写一个测试，实现它，然后继续
3. **重构安全网** - 测试是重构的保障
4. **边界条件优先** - 先测试边界情况，再测试正常流程
5. **测试即文档** - 好的测试可以替代文档
6. **持续重构** - 不要等待"大重构"，持续改进代码质量
